# Как подключить `SummonersExtention.d`

## Главное: для теста компиляция **не нужна**

В «Новом балансе» стоит **Union + zParserExtender**. Он подхватывает **`.d` прямо из `system\Autorun`** при запуске игры — без GothicSourcer и без `.dat`.

---

## Способ 1 — быстрый тест (рекомендуется)

1. Скопируйте файл:
   ```
   D:\Projects\Union\SummonersExtention\Scripts\SummonersExtention.d
   ```
   в:
   ```
   d:\Games\Steam\steamapps\common\Gothic II_8_0\system\Autorun\SummonersExtention.d
   ```

2. **Не** кладите туда `.dll` с другим именем — для DLL у вас уже VDF `NB_UNION_PLUGIN_ALLOWED_01`.

3. Запустите игру, загрузите сейв некроманта/призывателя.

4. Проверка (пока нет диалога учителя):
   - **Не через Marvin** — `SE_LearnSummonSlot1()` там даст `unknown command`.
   - В игре нажмите **F11** (обрабатывает Union-DLL): вызовется скрипт + сообщение.
   - Либо позже — из диалога учителя.

5. Посмотрите лимит призывов в UI (панель призыва / статус, если есть).

Если игра не ругается при старте — файл **подхватился**.

---

## Способ 2 — GothicSourcer (если нужен `.dat` / VDF)

Нужен только когда хотите **упаковать скрипты в `.vdf`**, а не держать `.d` в Autorun.

1. Запустите:
   ```
   D:\Programs\Gothic_tools\GothicSourcer\GothicSourcer\GothicSourcer\GothicSourcer.exe
   ```

2. **Файл → Открыть проект** (или создайте проект **Gothic II Addon / G2A**).

3. Укажите корень игры или `_work`:
   ```
   d:\Games\Steam\steamapps\common\Gothic II_8_0\_work
   ```

4. Добавьте в дерево проекта `SummonersExtention.d` (из папки `Scripts`).

5. **Компиляция → Скомпилировать скрипты** (или F7 — зависит от версии GS).

6. Результат: `Gothic.dat` / `Game.dat` в `_work\Data\Scripts\_compiled\`.

7. Упакуйте в VDF (vdfsbuilder / VDFS Tool) путь вида:
   ```
   _work\Data\Scripts\_compiled\Gothic.dat
   ```
   или положите только ваш патч в Autorun как `.d` (проще).

> **Не пересобирайте весь AB_Scripts.vdf** без опыта — легко сломать мод. Для вашего патча достаточно **одного `.d` в Autorun**.

---

## Что именно «компилировать»

| Файл | Куда | Компиляция |
|------|------|------------|
| `SummonersExtention.d` | `system\Autorun\` | **Нет** (Union injection) |
| `SummonersExtention.dll` | VDF `NB_UNION_PLUGIN_ALLOWED_01` | **Да** — Visual Studio, G2A MD Release |
| Весь AB мода | `AB_Scripts.vdf` | Только если делаете большой патч NB |

---

## Если `.d` не работает

1. В `system\Autorun` должен быть **zParserExtender** (часть Union/NB).
2. В `SystemPack.ini` можно попробовать:
   ```ini
   ParserExtenderAlwaysOn = true
   ```
3. Уберите `Game_Init()` из `.d`, если игра падает при старте — напишите, подберём другой хук.

---

## Дальше по модулю

После того как `.d` грузится:

1. Убрать тестовый `Message::Info` из `Plugin.cpp` (не обязательно для скриптов).
2. Сделать **диалог учителя**, вызывающий `SE_LearnSummonSlot1()` / `SE_LearnSummonSlot2()`.
3. Сохранять флаги навыков в переменных героя (чтобы не терялись после загрузки).
